# 분석엔진 확장 설계

## 배경

`lib/analytics/index.ts`의 `runAnalysis()`가 항상 자동 실행하는 파이프라인(스키마 → 품질 →
수치/범주 요약 → 상관분석 → 이상치 → 그룹비교 → 시계열 → 차트추천 → 지도감지 → 인사이트) 중
네 개 모듈(상관분석, 그룹비교, 이상치탐지, 시계열)을 통계적으로 고도화한다. 새로운 분석 기법이나
AI 연동을 추가하는 것이 아니라, 기존 네 모듈 각각의 결과에 유의성/효과크기/신뢰도 등 추가 지표와
공공기관 보고서 수준의 해석 문구를 덧붙이는 것이 목표다.

- 항상 자동 실행(옵트인 탭 아님) — 기존 파이프라인 순서·호출 방식은 그대로 유지
- 순수 TypeScript로 직접 구현, 신규 통계 라이브러리 의존성 추가 없음 (`package.json`에 통계
  라이브러리가 전혀 없는 기존 스타일 유지)
- 각 수치는 반드시 한국어 해석 문구를 동반한다

## 아키텍처

```
lib/analytics/
├── statUtils.ts          [신규] 확률분포 계산 공유 모듈
├── regressionUtils.ts    [신규] 선형대수 공유 모듈 (VIF 전용)
├── groupComparator.ts    [수정] 효과크기 추가, statUtils로 리팩터
├── correlationAnalyzer.ts [수정] p-value, 카이제곱/Cramér's V, VIF 추가
├── outlierDetector.ts    [수정] Z-score 병합 + 신뢰도 필드
├── dateDetector.ts       [수정] MoM/YoY 변화율 추가
└── index.ts              [무변경 또는 최소 수정] 반환 타입 확장만 반영
```

`groupComparator.ts`에 이미 구현된 `logGamma`(Lanczos 근사), `betacf`(연분수),
`regularizedIncompleteBeta`, `tTestPValue`, `fTestPValue`를 `statUtils.ts`로 이동하고,
카이제곱 p-value 계산에 필요한 정칙화 불완전감마함수(`regularizedIncompleteGamma`)를
신규 추가한다. 확률분포 계산(statUtils)과 선형대수(regressionUtils)를 분리하는 이유는
재사용 맥락이 다르기 때문이다 — 전자는 여러 유의성 검정에 반복 사용되고, 후자는 VIF
하나에만 쓰인다. 하나로 묶으면 각 모듈의 책임 경계가 흐려진다.

## 모듈별 신규 필드 및 계산 로직

### correlationAnalyzer.ts

- **수치×수치 쌍**: 기존 `pearsonCoefficient` 결과에 `pValue`(t = r·√((n-2)/(1-r²)), df = n-2,
  `statUtils.tTestPValue` 재사용), `significant: pValue < 0.05`, `interpretation`(예: "상관계수
  0.72, p=0.003으로 통계적으로 유의한 강한 양의 상관관계") 추가.
- **범주×범주 쌍** (신규): 분할표(contingency table) 생성 → 카이제곱 통계량 계산 →
  `statUtils.chiSquarePValue(chi2, df)` → Cramér's V(`√(chi2/(n·min(r-1,c-1)))`)로 효과크기
  산출. `significant`, `interpretation` 포함.
- **VIF** (신규, 수치 컬럼 전체 대상): 각 수치 변수를 나머지 수치 변수들로 회귀
  (`regressionUtils.multipleRegressionRSquared`) → `VIF = 1/(1-R²)`. VIF>10이면
  "다중공선성 높음" 해석 문구. 기존 상관쌍 배열과 별개 필드로 반환.

### groupComparator.ts

- 2그룹(Welch t-test): Cohen's d = (mean1-mean2)/pooled SD →
  `effectSize: { type: "cohen_d", value, magnitude }` (0.2/0.5/0.8 기준 작음/중간/큼).
- 3그룹+(ANOVA): eta² = SS_between/SS_total →
  `effectSize: { type: "eta_squared", value, magnitude }` (0.01/0.06/0.14 기준).
- `interpretation`에 유의성과 효과크기를 함께 서술(예: "p<0.05로 유의하나 효과크기(d=0.18)는
  작아 실질적 차이는 크지 않음" — 큰 표본에서 유의하지만 효과가 작은 경우를 명시).

### outlierDetector.ts

- 기존 IQR 결과에 더해 컬럼별 평균/표준편차 기반 Z-score(|z|>3) 계산.
- IQR·Z-score 둘 다 이상치로 판정 → `reliability: "고신뢰"`, 하나만 → `"참고"`.
- `outlierIndices`를 신뢰도별로 분리(`highConfidenceIndices`, `referenceIndices`)해 반환.

### dateDetector.ts

- 데이터가 월별 이상 간격으로 정렬 가능하면 마지막 시점 기준 전월대비(MoM)·전년동월대비(YoY)
  변화율(%) 계산해 `momChange`, `yoyChange`(계산 불가 시 `null`) 필드 추가.
- 계절성 감지, STL/트렌드 분해는 명시적으로 범위 밖.

## 예외·엣지케이스 처리

- **VIF**: 수치 컬럼 2개 미만이면 빈 배열 반환(에러 아님). 유효 행 수가 변수 개수+1보다
  적으면 해당 변수는 `vif: null, concern: false`로 스킵.
- **카이제곱/Cramér's V**: 범주형 컬럼 2개 미만이면 빈 배열. 분할표 셀 중 기대빈도<5인 셀이
  전체의 20%를 넘으면 `reliable: false` 플래그와 "표본이 작아 참고용" 해석 문구로 대체.
- **Z-score**: 표준편차 0인 컬럼은 Z-score 스킵, IQR 결과만 사용.
- **Cohen's d / eta²**: 그룹 표본수 n<2 또는 분산 0이면 `effectSize: null`. 기존
  `MAX_GROUP_COUNT=20` 등 상한 로직 유지.
- **MoM/YoY**: 날짜 간격이 불규칙하거나 월별 데이터가 아니면 둘 다 `null` — 계절성과 마찬가지로
  억지로 계산하지 않음.
- 모든 신규 함수는 예외를 던지지 않고 계산 불가 상황을 `null`/빈 배열/플래그로 명시적으로
  표현한다 — 기존 코드 스타일(방어적 조건 분기, try/catch 미사용)과 일치.

## 테스트 전략

- `lib/analytics/statUtils.test.ts` (신규): 기존 `groupComparator.test.ts`의 t/F 분포 p-value
  테스트 케이스를 이관 + `regularizedIncompleteGamma`/`chiSquarePValue` 알려진 카이제곱표
  값과 대조하는 케이스 추가.
- `lib/analytics/regressionUtils.test.ts` (신규): 선형독립 변수 셋(VIF≈1 기대)과 강한
  다중공선성 변수 셋(VIF 높음 기대) 각각 케이스.
- `lib/analytics/correlationAnalyzer.test.ts` (확장): p-value 유의/비유의 케이스, 카이제곱
  독립성 케이스(연관 있음/없음), VIF 계산 케이스, 각 엣지케이스(컬럼 부족 등) 1건씩.
- `lib/analytics/groupComparator.test.ts` (확장): Cohen's d/eta² 계산값 검증 케이스,
  큰 표본+작은 효과크기 조합 케이스(해석 문구 검증).
- `lib/analytics/outlierDetector.test.ts` (확장): IQR·Z-score 교집합/단독 케이스별
  reliability 값 검증.
- `lib/analytics/dateDetector.test.ts` (확장): MoM/YoY 계산 가능/불가능 케이스.
- `lib/analytics/index.test.ts`: 위 신규 필드가 `runAnalysis()` 전체 파이프라인 결과에
  정상 반영되는지 통합 케이스 1건.
- 전체 실행: `npx vitest run` 통과, `npm run build` 통과 확인.
