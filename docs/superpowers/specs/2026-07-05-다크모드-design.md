# 다크모드 설계

## 배경
정책 분석 엔진(`C:\업무\분석툴`)은 현재 `app/layout.tsx`에서 `data-theme="light"`로 고정되어 있고,
`components/AppTheme.tsx`도 `<Theme theme={neutralTheme}>`에 `mode`를 전달하지 않는다(기본값 `system`이지만
`<html>`은 정적으로 `light`로 못박혀 있어 실질적으로 라이트 고정). 사용자가 토글로 전환 가능한 다크모드를
별도 기능으로 요청.

핵심 발견: 사용하는 컴포넌트 라이브러리 `@astryxdesign/core`의 `Theme`는 `mode: 'light'|'dark'|'system'`를
1급으로 지원하고, `@astryxdesign/theme-neutral`은 이미 라이트/다크 양쪽 토큰(WCAG AA 대비 확보된
grayscale + 카테고리 색상)을 전부 정의해 두었다. 즉 **새 색상 팔레트를 정의할 필요가 없고**, 이미 있는
`mode` 스위치를 실제로 켜서 사용자가 제어할 수 있게 배선하는 작업이 전부다.

## 범위
- 포함: 모드 상태 관리(라이트/다크/시스템 3단 순환), localStorage 영속화, 첫 페인트 깜빡임(FOUC) 방지,
  사이드 내비게이션에 토글 UI 추가.
- 제외: 커스텀 다크 색상 팔레트 정의(불필요 — neutralTheme 재사용), 페이지별 다크모드 예외 처리(전역 1곳에서 제어).

## 아키텍처

### 1. 모드 상태 — `components/AppTheme.tsx`
`"use client"` 컴포넌트로 유지하되 상태를 갖도록 확장:
- `type ThemeModeSetting = "light" | "dark" | "system"`
- `useState`로 `mode` 보유, 초기값은 `localStorage.getItem("pae-theme-mode")` (없으면 `"system"`)
- `<Theme theme={neutralTheme} mode={mode}>`로 실제 모드 전달
- `React.createContext`로 `{ mode, cycleMode }`를 하위 트리에 노출(`ThemeModeContext`) — 사이드 내비의
  토글 버튼이 이 컨텍스트를 소비해 모드를 바꿈
- `cycleMode`: `light → dark → system → light` 순환, 호출 시 `localStorage.setItem("pae-theme-mode", next)`도 수행

### 2. FOUC 방지 — `app/layout.tsx`
현재 `<html lang="ko" data-theme="light">`로 정적 고정된 부분을 제거하고, `<head>`에 하이드레이션 이전에
동기 실행되는 인라인 스크립트(`dangerouslySetInnerHTML`)를 추가:
```js
(function(){
  try {
    var m = localStorage.getItem('pae-theme-mode');
    var dark = m === 'dark' || (m !== 'light' && window.matchMedia('(prefers-color-scheme: dark)').matches);
    document.documentElement.setAttribute('data-theme', dark ? 'dark' : 'light');
  } catch (e) {}
})();
```
이렇게 하면 Astryx의 `Theme`가 하이드레이션 후 자체적으로 `data-theme`을 다시 동기화하기 전까지도
첫 페인트가 저장된 선호와 일치한다. Astryx가 이미 문서화한 패턴("For RSC/SSR, set `data-theme` on
`<html>` in your root server layout to avoid a flash")을 인라인 스크립트로 동적 대응하는 것.

### 3. 토글 UI — `components/layout/AppNavShell.tsx`
사이드 내비게이션 하단(기존 "엔진" 그룹 아래)에 아이콘 버튼 추가:
- `lucide-react`의 `Sun`/`Moon`/`Monitor` 아이콘을 현재 모드에 따라 표시
- 클릭 시 `ThemeModeContext`의 `cycleMode()` 호출
- Astryx `Tooltip`으로 현재 모드 레이블 표시("라이트 모드"/"다크 모드"/"시스템 설정 따름")
- 기존 내비 항목과 동일한 시각 언어(아이콘 + 텍스트) 유지, 새 색상/그라디언트 도입 없음

## 데이터 흐름
```
localStorage("pae-theme-mode")
   ↓ (읽기, 최초 마운트 + 인라인 스크립트)
AppTheme state (mode)
   ↓ (prop)
Astryx <Theme mode={mode}> → data-theme on <html>, CSS 토큰 해석
   ↓ (context)
사이드 내비 토글 버튼 → cycleMode() → state 갱신 + localStorage 쓰기
```

## 에러 처리
- `localStorage` 접근 불가 환경(프라이빗 모드 등)에서는 `try/catch`로 무시하고 `system` 기본값 유지 —
  기능 저하는 있어도 크래시 없음.
- `window.matchMedia` 미지원 구형 브라우저: `system` 모드가 항상 `light`로 폴백(Astryx `useTheme`의
  `useMediaQuery` 훅 자체 동작 그대로 위임, 별도 처리 불필요).
